## Шаг 1
* **read** - читает данные со стандартного ввода и записывает их в переменные shell
  
* **decare** - задаёт свойства переменной
   * Флаг **i** - объявляет переменную целочисленной 

* **test** - встроенная команда для проверки условий: строковых, числовых и файловых, возвращающая код завершения

* **[** - короткая форма команды test (эквивалетны), задается как [условие]

* **if** - оператор ветвления, где после if стоит список команд. Если последняя команда вернула код 0, ветка считается истинной и выполняется then

* **select** - конструкция для простого пользовательского текстового меню

* **while** - цикл с предусловием: команды внутри цикла выполняются, пока последняя команда в условии возвращает код 0
  
* **for** - цикл перебора: переменная по очереди принимает значения из списка слов; если список не указан, берутся позиционные параметры сценария

Сценарий read_var.sh
```bash

```

Команда echo $ab выводит значение переменной ab. Так как переменная ab не была определена, то на экран выводится пустая строка.

Сценарий int_var.sh с помощью declare
```bash
nozdryakov@debian:~$ cat int_var.sh
ekate@DESKTOP-SMCGLNG:~$ nano int_var.sh
ekate@DESKTOP-SMCGLNG:~$ cat int_var.sh
#!/bin/bash

declare -i int_var

read -p "Введите значение: " int_var

echo "int_var = $int_var"
ekate@DESKTOP-SMCGLNG:~$
```

 - Ввод целого числа
```bash
ekate@DESKTOP-SMCGLNG:~$ ./int_var.sh
Введите значение: 20
int_var = 20
```

Если ввести целое число, то в int_var выведет это число.

 - Ввод строки
```bash
ekate@DESKTOP-SMCGLNG:~$ ./int_var.sh
Введите значение: lab6
int_var = 0
```

При вводе строки shell интерпретирует её как имя переменной. Если такая переменная не определена, в int_var записывается 0.

 - Ввод вещественного числа:
```bash
ekate@DESKTOP-SMCGLNG:~$ ./int_var.sh
Введите значение: 3.14
./int_var.sh: line 5: read: 3.14: syntax error: invalid arithmetic operator (error token is ".14")
int_var =
```

Если ввести вещественное число, то получаем ошибку арифметического синтаксиса, так как такое значение не является целым числом.

## Шаг 3 
Cценарий print_args.sh

```bash
ekate@DESKTOP-SMCGLNG:~$ nano print_args.sh
ekate@DESKTOP-SMCGLNG:~$ cat print_args.sh
#!/bin/sh

echo "Количество аргументов: $#"

echo "Аргументы через \$*: $*"
echo "Аргументы через \$@: $@"
```

```bash
ekate@DESKTOP-SMCGLNG:~$ ./print_args.sh lab6
Количество аргументов: 1
Аргументы через $*: lab6
Аргументы через $@: lab6
```

 - Имя запускаемого скрипта хранится в отдельном специальном параметре `$0` и не включается в `${*}`, `${@}` и `${#}`
 - `$#` показывает количество аргументов командной строки без учёта имени скрипта, тогда как argc в функции main языка Си включает в подсчёт и имя программы.

Добавлен в print_args.sh вывод позиционных параметров с именами 0 и 1

```bash
ekate@DESKTOP-SMCGLNG:~$ nano print_args.sh
ekate@DESKTOP-SMCGLNG:~$ cat print_args.sh
#!/bin/sh

echo "Количество аргументов: $#"

echo "Аргументы через \$*: $*"
echo "Аргументы через \$@: $@"

echo "Параметр 0: $0"
echo "Параметр 1: $1"
```

```bash
ekate@DESKTOP-SMCGLNG:~$ ./print_args.sh
Количество аргументов: 0
Аргументы через $*:
Аргументы через $@:
Параметр 0: ./print_args.sh
Параметр 1:
```

При запуске сценария без параметров: `$0` содержит имя запускаемого сценария, а `$1` пуст, поскольку первый аргумент не был передан

## Шаг 4
Сценарий root_test.sh

```bash
ekate@DESKTOP-SMCGLNG:~$ nano root_test.sh
ekate@DESKTOP-SMCGLNG:~$ cat root_test.sh
#!/bin/bash

if test -z "$1"
then
    echo "Отсутствует имя"
else
    if test "$1" = "root"
    then
        echo "Вы - суперпользователь"
    else
        echo "Вы - обычный пользователь"
    fi
fi
```

```bash
ekate@DESKTOP-SMCGLNG:~$ ./root_test.sh

Вы - суперпользователь
ekate@DESKTOP-SMCGLNG:~$ ./root_test.sh ekate
Вы - обычный пользователь
```

Cценарий poll_menu.sh
```bash
ekate@DESKTOP-SMCGLNG:~$ nano poll_menu.sh
ekate@DESKTOP-SMCGLNG:~$ cat poll_menu.sh
#!/bin/bash

echo "Выберите партию для голосования:"

select party in "ЯГОДА" "КЛМН" "МЕДРО" "ВВЖ" "СССР" "ПДД"
do
    if test -n "$party"
    then
        echo "Вы выбрали: $party"
        echo "Спасибо за участие в голосовании!"
        break
    else
        echo "Неверный выбор. Пожалуйста, выберите номер от 1 до 6."
    fi
done
```

```bash
2) КЛМН
3) МЕДРО
4) ВВЖ
5) СССР
6) ПДД
#? 1
Вы выбрали: ЯГОДА
Спасибо за участие в голосовании!
```
