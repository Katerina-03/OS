## Шаг 1 (справка)
1) **ps** - показывает информацию о процессах (можно посмотреть список запущенных процессов, узнать PID процесса, увидеть, кому принадлежит процесс)

2) **top** - показывает процессы в реальном времени в интерактивном режиме (можно сортировать процессы по CPU, памяти, PID и другим параметрам, завершать процессы прямо из интерфейса и т.д.)

3) **pstree** - показывает процессы в виде дерева (полезно для определения связи родительских и дочерних процессов)

4) **kill** - посылает процессу сигнал
   - Флаг **-l** - показывает список сигналов с их именами и номерами
   - Флаг **-s** - позволяет явно указать, какой сигнал отправить процессу
   - Флаг **-9** - принудительно завершает процесс без возможности обработки этого сигнала программой (используют, когда процесс не завершается обычным способом)

5) **bg** - переводит приостановленную задачу в фоновый режим

6) **fg** - переводит фоновую или остановленную задачу обратно в интерактивный режим

1.4. **Сравните вывод команды из пункта 1.3 с результатом вывода команды ps -f -u $(whoami). В чем состоит отличие?**
```bash
nozdryakov@debian:~$ ps -f -u $(whoami)
UID          PID    PPID  C STIME TTY          TIME CMD
nozdrya+   96494       1  0 19:39 ?        00:00:00 /lib/systemd/systemd --user
nozdrya+   96496   96494  0 19:39 ?        00:00:00 (sd-pam)
nozdrya+   96515   96491  0 19:39 ?        00:00:00 sshd: nozdryakov@pts/0
nozdrya+   96516   96515  0 19:39 pts/0    00:00:00 -bash
nozdrya+   96535   96516  0 19:48 pts/0    00:00:00 ps -f -u nozdryakov
```

**Отличие** заключается в том, что в **варианте 1.3** идет текстовый поиск по полному списку процессов, поэтому показывает только те процессы, в описании которых встречается имя моего пользователя, то есть nozdryakov. Именно поэтому были показаны:
  - `sshd: nozdryakov [priv]`
  - `sshd: nozdryakov@pts/0`
  - `grep nozdryakov`

Но не показались другие процессы моего пользователя, например эти:
  - `/lib/systemd/systemd --user`
  - `(sd-pam)`
  - `bash`

Они тоже принадлежат моему пользователю, но в строке процесса нет текста nozdryakov, поэтому grep их не нашёл.

В **варианте 1.4** отбор идёт не по тексту строки, а по владельцу процесса. Поэтому команда показывает все процессы текущего пользователя, даже если имя пользователя нигде не написано в поле.

**Еще отличие** заключается в том, что в 1.3 появился лишний процесс: `grep nozdryakov`. Это сам grep, потому что в момент выполнения он тоже был процессом, и его командная строка содержит nozdryakov. В 1.4 такого лишнего процесса нет, потому что ps сам отбирает процессы пользователя, а не ищет совпадение по тексту.
