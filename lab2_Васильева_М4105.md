## Шаг 1

#### `time` - Замеряет и показывает время выполнения команды (real, user, sys).

#### `find` - Ищет файлы и папки по заданным критериям (имя, размер, дата).

#### `cut` - Вырезает нужные колонки или символы из строк текста.

#### `sort` - Сортирует строки в алфавитном или числовом порядке.

#### `uniq` - Убирает повторяющиеся строки (обычно используется после `sort`).

#### `tee` - Читает ввод и пишет результат одновременно в файл и на экран.

#### `nl` - Выводит содержимое файла, нумеруя строки.

#### `du` - Показывает, сколько места на диске занимают файлы и папки.

#### `uname` - Выводит информацию о системе (ядро, архитектура, имя узла).

#### `mesg` - Разрешает или запрещает отправку сообщений на твой терминал.

#### `write` - Отправляет сообщение другому пользователю в реальном времени.

## Шаг 2

* Подсчитать количество всех объектов (включая скрытые файлы) в текущем каталоге - `ls -1A | wc -l`. Вызов без `-1A` не учитывает скрытые файлы.

* Выполнить поиск файла README от корневого каталога и подсчитать затраченное на поиск - `time find / -name "README" 2>/dev/null`.

* Вывести входные уникальные имена всех пользователей, работающих в системе - `who | cut -d' ' -f1 | sort | uniq`.

* Вывести содержимое файла commands в файл commands2 и на экран с нумерацией непустых строк - `cat -n commands | tee commands 2`

``` 
     1  ls -A1 | wc -l 
     2  time find / -name "README" 2>/dev/null 
     3  who | cut -d  -f1 | sort | uniq 
```

Команда `time` используется для измерения времени выполнения другой команды.
`time <строка-команда>`  - измеряет время выполнения строки-команды.
`time ! <cтрока-команда>` - измеряет время выполнения предыдущей команды в истории команд.

## Шаг 3
* Вывести полную информацию о текущем каталоге - `ls -ld`.

Данная команда содержит информацию о правах доступа к каталогу, количестве ссылок на него, его группе и владельце, размере, времени последней модификации.
Если имя каталога содержит пробельные символы, то команда `ls` будет интерпретировать каждое слово в имени как отдельный аргумент.
Команда `ls` ожидает один аргумент - имя каталога. Чтобы исправить ошибку, нужно заключить имя каталога в двойные кавычки.

* Вывести размер всех файлов в домашнем каталоге, которые являтся директориями. Вывод ошибок команды find необходимо подавить - `find ~ -type d -exec du -sh {} \; 2>/dev/null`

Если имя каталога содержит пробельные символы, команда `du` выдает ошибку «Нет такого файла или каталога». 
Это происходит потому, что команда `du` разделяет аргументы по пробельным символам, и если имя каталога содержит пробелы, команда `du` будет 
интерпретировать каждую часть имени как отдельный файл или каталог.

## Шаг 4

* Вывод информации о системе в файл sys.info - `(uname -s; uname -n; uname -m; uname -r) > sys.info`:
  * `uname -s` - выводит имя ядра (например, Linux)
  * `uname -n` - выводит имя узла (hostname)
  * `uname -m` - выводит архитектуру машины (например, x86_64)
  * `uname -r` - выводит версию ядра (например, 5.4.0-42-generic)

*  Проверка возможности отправки сообщения на экран - `mesg >/dev/null && echo "Сообщения разрешены" || echo "Сообщения запрещены"`.

  **Как это работает:**
- `mesg` без аргументов возвращает код:
  - `0` (успех)
  - `1` (ошибка)
- `>/dev/null` — скрывает вывод `mesg`
- `&&` — выполняется, если код `0`
- `||` — выполняется, если код `1`
