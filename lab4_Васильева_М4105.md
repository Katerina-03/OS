## Шаг 1 

+ **ps** - показывает информацию о процессах

+ **top** - показывает процессы в реальном времени в интерактивном режиме

+ **pstree** - показывает процессы в виде дерева

+ **kill** - посылает процессу сигнал
   - **-l** - показывает список сигналов с их именами и номерами
   - **-s** - позволяет явно указать, какой сигнал отправить процессу
   - **-9** - принудительно завершает процесс без возможности обработки этого сигнала программой

+ **bg** - переводит приостановленную задачу в фоновый режим

+ **fg** - переводит фоновую или остановленную задачу обратно в интерактивный режим

## Шаг 3

Запуск на выполнение команды find / -name "*" >/dev/null**
```bash
ekate@DESKTOP-SMCGLNG:~$ find / -name "*" -exec echo {} \; >/dev/null 2>/dev/null
```

В другом терминале определен PID запущенного процесса и отправлен ему сигнал SIGTSTP. Объясните, что произойдет
```bash
ekate@DESKTOP-SMCGLNG:~$ ps -Af | grep find
ekate        938     320 15 09:23 pts/0    00:00:02 find / -name * -exec echo {} ;
ekate      21449    6972  0 09:24 pts/2    00:00:00 grep --color=auto find
ekate@DESKTOP-SMCGLNG:~$ kill -s SIGTSTP 938
```

После отправки сигнала SIGTSTP процесс не завершается, а только приостанавливается. Его выполнение временно останавливается, позже этот процесс можно продолжить.

```bash
ekate@DESKTOP-SMCGLNG:~$ kill -l SIGTSTP
20
```

Сигнал SIGTSTP соответствует номеру 20.

В первом терминале отправлен приостановленному процессу сигнал на завершение (SIGKILL) и выведен код возврата (echo $?)
```bash
ekate@DESKTOP-SMCGLNG:~$ kill -9 938
ekate@DESKTOP-SMCGLNG:~$ echo $?
0
[9]+  Killed                  find / -name "*" -exec echo {} \; > /dev/null 2> /dev/null
```

 Значение 0 означает, что сигнал был успешно отправлен. Сообщение показывает, что процесс find действительно был принудительно завершён сигналом SIGKILL

## Шаг 4

Запуск в фоновом режиме редактор emacs
```bash
ekate@DESKTOP-SMCGLNG:~$ emacs file &
[9] 80783
```

C помощью команды jobs получен список запущенных в терминале задач
```bash

[1]-  Stopped     top

```

Запуск комады man ls и выполнение терминальную остановку с помощью Ctrl+Z
```bash
ekate@DESKTOP-SMCGLNG:~$ man ls

[3]+  Stopped    man ls
```

Отображене списока задач
```bash
ekate@DESKTOP-SMCGLNG:~$ jobs

[1]   Stopped    top
[2]-  Stopped    emacs file
[3]+  Stopped    man ls
```

Перевод второй задачи в фоновый режим (bg)
```bash
ekate@DESKTOP-SMCGLNG:~$ bg %2
[2]- emacs file &
```

Перевод первой задачи в интерактивный режим (fg)
```bash
ekate@DESKTOP-SMCGLNG:~$ fg %1
```

Удаление задач с помощью команды kill
```bash
ekate@DESKTOP-SMCGLNG:~$ kill %1
-bash: kill: %1: no such job

ekate@DESKTOP-SMCGLNG:~$ kill %2

[2]+  Stopped   emacs file

ekate@DESKTOP-SMCGLNG:~$ kill %3

[3]   Stopped      man ls
```

Для задачи 1 было получено: "kill: %1: no such job", поскольку после fg %1 и выхода из top она завершилась и перестала быть заданием оболочки.

**Для запуска задачи в фоновом режиме** в конце команды ставится символ &. Например: emacs file &.
**Для перевода задачи из фонового режима в интерактивный** используется команда fg с указанием номера задачи, например: fg %1.
